把整站塞进一个 HTML 文件，听起来像过时的小实验。但当你真正这样做时，会重新感受到网页最原始的魅力：打开即所得，没有构建步骤，也没有依赖迷宫。

## 边界很清晰

单文件适合：

- 落地页、作品展示、小型工具
- 需要极致可移植性的页面
- 想把设计与实现压到同一视线里的原型

它不适合：

- 持续增长的多篇文章站点
- 需要复用布局、组件和内容集合的长期项目

这也是本站迁到 Eleventy 的原因——保留极简视觉，同时让写作以 Markdown 自然生长。

## 为什么仍然迷人

约束会 sharpen 判断。没有组件库时，你只能认真对待层级；没有打包器时，你只能认真对待体积。很多“现代化”复杂度，其实并不是表达所必需的。

单文件不是终点，而是一种提醒：复杂应当被挣来，而不是默认配发。
