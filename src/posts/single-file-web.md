把一个完整页面收进单个 HTML 文件，听起来像过时的小实验。但真正动手时，网页最朴素的魅力会重新出现：文件双击就能打开，复制到哪里都能继续工作。

## 边界很清晰

单文件适合：

- 落地页、作品展示、小型工具
- 需要极致可移植性的页面
- 想在同一处同时处理设计与实现的原型

它不适合：

- 持续增长的多篇文章站点
- 需要复用布局、组件和内容集合的长期项目

这也是本站迁到 Eleventy 的原因——保留极简视觉，同时让写作以 Markdown 自然生长。

## 为什么仍然迷人

约束会让判断变得具体。没有组件库，就得亲自处理层级；没有打包器，就会自然留意体积。许多复杂度只是工具默认附带，与页面真正要做的事无关。

单文件不是终点，只是在提醒我们：增加复杂度之前，最好先说清楚理由。
